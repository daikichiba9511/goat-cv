# ADR-0001: Go HTTPフレームワーク

## Status

Accepted

## Context

Go製バックエンドのHTTPフレームワークを選定する。
将来的にWebSocket対応（共同編集）が必要になるため、拡張性とロックインの低さが重要。

## Options

### Chi

- 標準`net/http`互換のルーター/ミドルウェア
- `http.Handler`をそのまま使えるため、標準ライブラリの知識が活きる
- ミドルウェアが標準互換で再利用しやすい
- バリデーション等は自前 or 別ライブラリ

### Echo

- 独自`echo.Context`による高機能フレームワーク
- バリデーション、バインディング組み込み
- 独自ContextのためミドルウェアはEcho専用が必要
- フレームワークへのロックイン度が高い

### Gin

- 最もスター数が多い（~80k）、コミュニティ最大
- 独自`gin.Context`
- パフォーマンス重視の設計
- Echo同様、フレームワーク固有のロックインがある

## Decision

**Chi** を採用する。

## Rationale

- 標準`net/http`互換で**ロックインが最も低い**
- 将来WebSocketを足す際も標準のHTTPハンドラとして自然に統合できる
- 小さく始めて必要に応じてミドルウェアを足す開発スタイルに合う
- バリデーションが組み込みでない点は、境界層でpydantic的なバリデーションを自前で行う方針と一致する
